# Knee/Shoulder Monitoring System

## 1. 프로젝트 한 줄 정의

매일 종가 기준으로 종목별 `무릎 후보(knee)`와 `어깨 후보(shoulder)`를 탐지하고, 다음날 이후 실제 성과까지 검증해서 누적 승률을 보여주는 모니터링 시스템.


## 2. 핵심 목표

- 사람이 모든 차트를 직접 보지 않아도 매일 주목할 종목을 자동으로 추린다.
- "정확한 바닥/천장 예측"이 아니라 "반전 가능성이 높아진 후보 탐지"에 집중한다.
- 신호 발생 후 다음날, 3일, 5일, 10일 성과를 자동 기록해 시스템 성능을 검증한다.


## 3. 최소 필요 데이터

필수 일봉 데이터:

- `date`
- `open`
- `high`
- `low`
- `close`
- `volume`

있으면 좋은 데이터:

- `turnover`(거래대금)
- `pct_change`
- `market_cap`
- `sector`
- 시장지수 데이터


## 4. 주요 기능

### 4.1 데이터 관리

- 종목별 일봉 데이터 저장
- 일 단위 데이터 업데이트
- 누락 데이터 확인
- 종목 마스터 관리

### 4.2 신호 계산

- `knee_score` 계산
- `shoulder_score` 계산
- 신호 발생 이유 저장
- 신호 등급 분류
  - Strong
  - Watch
  - Neutral

### 4.3 검증 기능

- 신호 발생일 저장
- 다음날 결과 계산
- 3일, 5일, 10일 결과 계산
- 최대 상승폭(MFE), 최대 하락폭(MAE) 기록
- 성공/실패 판정

### 4.4 조회 기능

- 오늘 무릎 후보 리스트
- 오늘 어깨 후보 리스트
- 종목 상세 화면
- 과거 신호 이력 조회
- 누적 통계 조회


## 4A. 우려 반영 설계 원칙

이 시스템은 아래 우려를 전제로 설계한다.

- 무릎/어깨는 주관적이라 정답이 애매하다.
- 가짜 반전 신호가 많다.
- 과거 데이터에만 맞춘 과최적화가 발생할 수 있다.
- 시장 상황이 바뀌면 같은 규칙의 성능이 달라진다.
- 거래량이 적은 종목은 신호 신뢰도가 낮다.

따라서 아래 원칙을 기본 구조에 포함한다.

### 4A.1 확정 판정 금지

- `무릎이다`, `어깨다`로 확정하지 않는다.
- 항상 `knee_score`, `shoulder_score`로 점수형 표시를 한다.
- 점수와 함께 이유를 반드시 저장한다.

### 4A.2 2단계 신호 구조

- 1단계: 후보 탐지
- 2단계: 확인 신호

예시:

- 무릎 후보: 저점권 + RSI 반등 + 거래량 증가
- 무릎 확인: 20일선 회복 또는 전일 고점 돌파
- 어깨 후보: 고점권 + RSI 약화 + 거래량 이상
- 어깨 확인: 20일선 이탈 또는 전일 저점 이탈

이 구조를 통해 가짜 반전 신호를 줄인다.

### 4A.3 유동성 필터 기본 적용

- 최소 거래량 조건을 둔다.
- 최소 거래대금 조건을 둔다.
- 비정상 급등락, 초저유동성 종목은 기본 제외한다.

### 4A.4 검증 중심 구조

- 신호 생성보다 신호 검증을 더 중요한 기능으로 둔다.
- 모든 신호는 다음날, 3일, 5일, 10일 성과를 자동 기록한다.
- 성공/실패뿐 아니라 MFE, MAE도 같이 저장한다.

### 4A.5 과최적화 방지

- 첫 버전은 단순 규칙으로 시작한다.
- 규칙 수를 과도하게 늘리지 않는다.
- 점수 산식 변경 전후 성능을 비교한다.
- 검증은 전체기간 평균만 보지 않고 기간을 나눠서 본다.

### 4A.6 시장 상태 분리

- 상승장
- 하락장
- 횡보장

위 3개 상태에서 신호 성능을 따로 본다.

### 4A.7 설명 가능성 확보

- 점수만 보여주지 않는다.
- 사용자 화면에 "왜 이 신호가 나왔는지"를 짧고 명확하게 표시한다.
- 예: `최근 20일 저점권`, `RSI 반등`, `거래량 1.8배`, `20일선 회복`


## 5. 기본 판단 로직

첫 버전은 너무 복잡하게 가지 않고 아래처럼 구성한다.

무릎 후보 판단 예시:

- 최근 20일 또는 60일 저점권 근처
- 하락 후 종가 반등
- RSI 반등 또는 MACD 개선
- 거래량 평균 대비 증가
- 20일선 재돌파 또는 이탈 후 회복

어깨 후보 판단 예시:

- 최근 20일 또는 60일 고점권 근처
- 상승 후 종가 약세 전환
- RSI 약화 또는 MACD 둔화
- 고점권 거래량 이상 신호
- 20일선 이탈 또는 단기 지지 붕괴

중요 원칙:

- 신호는 "예측"이 아니라 "후보 탐지"로 본다.
- 점수는 0~100으로 계산한다.
- 점수와 함께 반드시 이유를 저장한다.
- 후보 신호와 확인 신호를 분리한다.
- 유동성 필터를 통과한 종목만 평가한다.


## 6. 점수 예시 구조

### 6.1 Knee Score

- 최근 20일 저점권 근접: +20
- RSI 반등: +15
- MACD 개선: +15
- 거래량 증가: +15
- 20일선 회복: +20
- 장기 하락 과도: +15

### 6.2 Shoulder Score

- 최근 20일 고점권 근접: +20
- RSI 약화: +15
- MACD 둔화: +15
- 거래량 이상 징후: +15
- 20일선 이탈: +20
- 단기 급등 과열: +15

점수 해석 예시:

- `80~100`: 강한 후보
- `65~79`: 관심 후보
- `0~64`: 관찰만


## 7. 검증 로직 예시

무릎 성공 예시:

- 신호 후 5거래일 안에 `+3%` 이상 상승
- 또는 3거래일 평균 수익률이 플러스

어깨 성공 예시:

- 신호 후 5거래일 안에 `-3%` 이상 하락
- 또는 3거래일 평균 수익률이 마이너스

추가 검증 항목:

- 신호 다음날 수익률
- 3일 뒤 수익률
- 5일 뒤 수익률
- 10일 뒤 수익률
- 최대 유리폭(MFE)
- 최대 불리폭(MAE)

검증 시 추가로 반드시 볼 항목:

- 점수 구간별 승률
- 시장 상태별 승률
- 거래량 필터 포함/제외 성능 차이
- 최근 1개월과 과거 6개월 성능 차이


## 7A. 운영 리스크 대응 기능

실전 운영 시 아래 기능을 기본 포함한다.

- 점수 산식 버전 관리
- 신호 발생 당시 원인 로그 저장
- 검증 기준 변경 이력 저장
- 최근 성능 악화 시 경고 표시

예시:

- 최근 20거래일 무릎 승률이 기준 이하로 하락
- 특정 시장 상태에서 어깨 신호 성능 급락
- 거래량 필터 제거 시 오탐 증가

이 기능은 시스템이 틀리고 있는 상태를 빨리 발견하기 위해 필요하다.


## 8. 추천 화면 구조

화면은 4개로 나눈다.

1. 메인 대시보드
2. 종목 상세
3. 검증 통계
4. 설정


## 9. 화면 1: 메인 대시보드

### 9.1 목적

오늘 가장 먼저 볼 종목과 조심할 종목을 한 화면에서 파악한다.

### 9.2 상단 요약 카드

- 오늘 무릎 강한 후보 수
- 오늘 어깨 강한 후보 수
- 최근 20거래일 무릎 승률
- 최근 20거래일 어깨 승률
- 오늘 신규 신호 수
- 평균 거래대금 필터 통과 종목 수

예시:

```text
+------------------+  +------------------+  +------------------+
| Knee Strong 12   |  | Shoulder Strong 9|  | New Signals 21   |
| Top reversal long|  | Top reversal risk|  | Today detected   |
+------------------+  +------------------+  +------------------+

+------------------+  +------------------+  +------------------+
| Knee Win 58%     |  | Shoulder Win 54% |  | Filtered 312     |
| recent 20 days   |  | recent 20 days   |  | liquid names     |
+------------------+  +------------------+  +------------------+
```

### 9.3 메인 본문 레이아웃

좌측에는 무릎 후보, 우측에는 어깨 후보를 둔다.

```text
+------------------------------------------------+------------------------------------------------+
| Knee Candidates                                | Shoulder Candidates                            |
|------------------------------------------------|------------------------------------------------|
| 종목명 | 현재가 | Knee | Vol | 이유요약         | 종목명 | 현재가 | Shoulder | Vol | 이유요약     |
| A     | 52,100 | 88   | 1.8x| 저점권+RSI반등   | X     | 110,500| 84       | 2.2x| 고점권+약세  |
| B     | 18,400 | 81   | 1.5x| 이평회복+거래량  | Y     | 72,000 | 79       | 1.7x| MACD둔화     |
| C     | 96,300 | 77   | 2.1x| MACD개선         | Z     | 44,200 | 75       | 1.4x| 지지이탈     |
+------------------------------------------------+------------------------------------------------+
```

### 9.4 하단 보조 영역

- 최근 20일 승률 추이 그래프
- 점수 구간별 성능 요약
- 오늘 전체 시장 분포

예시:

```text
+--------------------------------------------------------------------------------------+
| Recent Performance                                                                   |
| Knee win-rate trend | Shoulder win-rate trend | Score buckets                        |
+--------------------------------------------------------------------------------------+
```

### 9.5 추천 컬럼

무릎/어깨 후보 표 공통 컬럼:

- 종목명
- 현재가
- 당일 등락률
- `knee_score` 또는 `shoulder_score`
- 거래량 배수(`today_volume / avg20_volume`)
- 최근 고점/저점 대비 위치
- 이유 요약
- 신호일


## 10. 화면 2: 종목 상세

### 10.1 목적

왜 이 종목이 무릎 또는 어깨 후보인지 차트와 지표로 바로 확인한다.

### 10.2 화면 구성

```text
+--------------------------------------------------------------------------------------+
| 종목명 / 현재가 / 당일등락 / Knee Score / Shoulder Score                             |
+--------------------------------------------------------------------------------------+
| Price Chart                                                                          |
| - 캔들 또는 선차트                                                                   |
| - 20일선, 60일선, 120일선                                                            |
| - 과거 신호 발생 지점 마커                                                           |
+--------------------------------------------------------------------------------------+
| Volume Chart                                                                         |
| - 거래량 막대                                                                         |
| - 20일 평균 거래량 선                                                                 |
+--------------------------------------------------------------------------------------+
| Signal Reasons                         | Validation Snapshot                         |
|----------------------------------------|---------------------------------------------|
| 최근 20일 저점권                       | 최근 무릎 신호 5회 중 성공 3회              |
| RSI 반등                               | 평균 5일 성과 +2.8%                          |
| MACD 시그널 개선                       | 평균 MFE +4.1%                               |
| 거래량 1.8배                           | 평균 MAE -1.6%                               |
+--------------------------------------------------------------------------------------+
```

### 10.3 상단 핵심 정보

- 종목명
- 티커
- 현재가
- 등락률
- `knee_score`
- `shoulder_score`
- 최근 신호 상태

### 10.4 차트에서 보여줄 요소

- 가격
- 이동평균선(20, 60, 120)
- 최근 신호 마커
- 거래량
- 신호 발생일 툴팁

### 10.5 하단 테이블

과거 신호 이력:

- 신호일
- 신호 유형
- 신호 점수
- 다음날 수익률
- 3일 뒤
- 5일 뒤
- 10일 뒤
- 성공/실패


## 11. 화면 3: 검증 통계

### 11.1 목적

시스템이 실제로 맞는지 숫자로 판단한다.

### 11.2 핵심 카드

- 전체 신호 수
- 무릎 승률
- 어깨 승률
- 평균 1일 성과
- 평균 5일 성과
- 최근 1개월 성과

### 11.3 주요 표/차트

```text
+--------------------------------------------------------------------------------------+
| Validation Summary                                                                   |
| Total signals | Knee win-rate | Shoulder win-rate | Avg 5D return | Avg MFE | Avg MAE|
+--------------------------------------------------------------------------------------+

+--------------------------------------+-----------------------------------------------+
| Score Bucket Analysis                 | Holding Period Analysis                       |
| 60~69 / 70~79 / 80+ win-rate          | 1D / 3D / 5D / 10D performance                |
+--------------------------------------+-----------------------------------------------+

+--------------------------------------------------------------------------------------+
| Condition Comparison                                                                  |
| 거래량 포함 조건 vs 제외 조건                                                         |
| 20일선 회복 포함 vs 제외                                                              |
+--------------------------------------------------------------------------------------+
```

### 11.4 보고 싶은 분석

- 점수 높을수록 승률이 올라가는지
- 거래량 조건이 성능을 올리는지
- 무릎은 며칠 보유할 때 가장 좋은지
- 어깨는 단기 하락 포착에 유리한지
- 상승장/하락장/횡보장 중 어디서 잘 맞는지
- 최근 성능이 과거 대비 나빠지고 있는지


## 12. 화면 4: 설정

### 12.1 목적

시스템 기준값을 쉽게 바꾼다.

### 12.2 항목

- 최소 거래량
- 최소 거래대금
- `knee_score` 알림 기준
- `shoulder_score` 알림 기준
- 검증 기간 선택
- 성공 판정 기준
- 시장 상태 분류 기준
- 후보/확인 신호 사용 여부
- 시장 구분
  - 한국
  - 미국
  - ETF

예시:

```text
[Filters]
- Min volume: 100,000
- Min turnover: 1,000,000,000
- Knee alert threshold: 75
- Shoulder alert threshold: 75
- Confirmation required: on
- Market regime filter: on

[Validation]
- Forward days: 1 / 3 / 5 / 10
- Knee success: +3% within 5 days
- Shoulder success: -3% within 5 days
```


## 13. 추천 사용자 흐름

### 아침 또는 장 마감 후

1. 메인 대시보드에서 오늘 강한 후보를 확인한다.
2. 무릎 점수 상위 종목 3~5개를 본다.
3. 어깨 점수 상위 종목 3~5개를 본다.
4. 관심 종목을 종목 상세 화면에서 차트로 확인한다.

### 다음날 이후

1. 검증 통계 화면에서 전일 신호 성과를 확인한다.
2. 최근 20거래일 승률이 유지되는지 본다.
3. 필요하면 설정 화면에서 기준을 조정한다.


## 14. 추천 UI 원칙

- 메인 화면은 한눈에 보이게 구성한다.
- 강한 후보만 먼저 드러낸다.
- 이유를 짧게 보여준다.
- 표와 차트를 같이 둔다.
- 색상은 단순하게 쓴다.

추천 색상:

- 무릎: 초록 계열
- 어깨: 빨강 계열
- 중립: 회색

추천 강조 방식:

- 점수 80 이상: 진한 색
- 점수 65~79: 중간 색
- 점수 64 이하: 기본색


## 15. 1차 개발 범위 제안

처음부터 모든 걸 만들지 말고 아래까지만 구현한다.

- 일봉 데이터 로드
- `knee_score`, `shoulder_score` 계산

## 16. 최종 정리

이 시스템은 자동매매 프로그램이 아니라 아래 목적의 모니터링 시스템이다.

- 오늘 바닥 반등 가능성이 커진 종목 찾기
- 오늘 고점 꺾임 가능성이 커진 종목 찾기
- 실제로 맞았는지 다음날 이후 검증하기
- 시간이 갈수록 어떤 조건이 잘 맞는지 학습하기

즉, "무릎과 어깨를 맞히는 도구"가 아니라 "무릎과 어깨 후보를 일관되게 추적하고 검증하는 도구"로 설계하는 것이 맞다.
